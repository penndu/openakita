本文用于手工验收「AIGC 视频创作工作室」组织模板，以及通义生图、即梦视频两个工作台节点在组织编排中的任务传递、附件登记和最终交付链路。

## 1. 测试目标

验证以下三条入口都能触发同一套组织协作流程：

1. 在组织编排页面的「指挥台」中直接下达任务。
2. 在主聊天窗口通过 `@org:组织名` 调用组织。
3. 通过 IM 通道（飞书、钉钉、Telegram 等即时通讯入口）向组织下达同类任务。

需要重点观察：

1. 制片人能否把任务拆给编剧、通义生图工作台、即梦视频工作台。
2. 通义生图工作台是否生成图片，并返回 `image_urls`、`local_paths`、`asset_ids`。
3. 组织运行时是否把图片下载到 org workspace，并登记为任务附件。
4. 制片人是否把上游 `asset_ids` 传给即梦视频工作台。
5. 即梦视频工作台是否通过 `from_asset_ids` 生成视频，并返回 `video_url`、`video_path`、`asset_ids`。
6. 最终交付是否包含脚本、分镜图、视频附件，而不是只返回一段纯文本。

## 2. 术语说明

- 组织编排：把多个 AI 节点组织成上下级或协作关系，让它们自动派工、交付和验收。
- 指挥台：组织页面里用户向整个组织发号施令的入口，相当于老板给团队下任务。
- 工作台节点：由插件映射而来的叶子节点。比如通义生图工作台调用 `tongyi-image` 插件，即梦视频工作台调用 `seedance-video` 插件。
- API：后端接口，前端页面通过它读取模板、保存组织、启动组织。
- IM：即时通讯通道，例如飞书、钉钉、Telegram、企业微信、QQ、OneBot。
- Asset Bus：插件之间传递图片、视频等产物的内部资产总线。通义生图产出的 `asset_ids` 可以被即梦视频的 `from_asset_ids` 消费。

## 3. 前置条件

### 3.1 必须安装并启用工作台插件

用户需要先安装好对应工作台应用，也就是两个插件都必须在「插件管理」中处于已安装、已启用、已加载状态：

1. `tongyi-image`
   - 用途：文生图 / 图生图。
   - 需要配置通义千问 / DashScope 相关 API Key，例如 `DASHSCOPE_API_KEY`。
   - 插件权限需要包含 `assets.publish`。

2. `seedance-video`
   - 用途：文生视频 / 图生视频。
   - 需要配置即梦 / Ark 视频相关 API Key，例如 `ARK_API_KEY` 或插件页面要求的同等凭证。
   - 插件权限需要包含 `assets.publish` 和 `assets.consume`。

验收点：

1. 打开「插件管理」。
2. 确认 `tongyi-image` 和 `seedance-video` 都显示为已启用。
3. 如果页面有工具列表，确认能看到：
   - `tongyi_image_create`
   - `tongyi_image_status`
   - `tongyi_image_list`
   - `seedance_create`
   - `seedance_status`
   - `seedance_list`

### 3.2 后端和前端启动

桌面版用户：

1. 启动 OpenAkita Desktop。
2. 确认后端状态正常。
3. 打开「组织编排」页面。

开发环境用户：

```bash
openakita serve
```

如果需要前端开发模式：

```bash
cd apps/setup-center
npm run dev:web
```

### 3.3 创建测试组织

1. 进入「组织编排」。
2. 点击「从模板创建」。
3. 选择「AIGC 视频创作工作室」。
4. 创建后确认画布中至少有 4 个节点：
   - 制片人
   - 编剧
   - 通义生图工作台
   - 即梦视频工作台
5. 确认两个工作台节点卡片上显示「工作台」徽章。
6. 点开通义生图工作台的能力配置页，确认工具列表只读。
7. 点开即梦视频工作台的能力配置页，确认工具列表只读。
8. 确认两个工作台节点没有下属节点。工作台节点必须是叶子节点，不能再挂子节点。

建议把组织名称改成固定名称，方便后续聊天和 IM 测试：

```text
AIGC 视频创作工作室测试
```

### 3.4 启动组织

1. 在组织页面点击「启动」。
2. 等待状态变成运行中。
3. 进入实况模式或打开组织消息侧边栏，方便观察任务流。

## 4. 测试数据准备

为了减少模型自由发挥导致的变量，本计划使用同一个短视频需求做三次入口测试。

测试主题：

```text
30 秒竖屏短片：未来城市咖啡广告
```

核心要求：

```text
风格：赛博朋克、霓虹灯、雨夜、电影感
画幅：9:16 竖屏
时长：约 30 秒
镜头数：3 个
产物：完整脚本、每个镜头的分镜图、最终视频
```

## 5. 路径一：组织指挥台测试

### 5.1 基础端到端测试

入口：

1. 打开「组织编排」。
2. 进入「AIGC 视频创作工作室测试」。
3. 打开指挥台输入框。

用户输入：

```text
请制作一个 30 秒竖屏 AIGC 短片，主题是「未来城市咖啡广告」。

要求：
1. 先让编剧产出完整脚本，包含 3 个镜头，每个镜头要有画面描述、旁白或字幕、镜头运动、时长。
2. 再把每个镜头的视觉描述交给通义生图工作台生成关键帧分镜图。
3. 通义生图工作台生成图片后，请把它返回的 asset_ids 传给即梦视频工作台。
4. 即梦视频工作台必须使用 seedance_create 的 from_asset_ids 参数消费这些图片，再生成视频。
5. 最终交付时请给我：脚本文本、分镜图附件、视频附件，并说明每个附件对应哪个镜头。

风格：赛博朋克、霓虹灯、雨夜、电影感。
画幅：9:16。
```

预期过程：

1. 制片人收到任务。
2. 制片人调用 `org_delegate_task` 把脚本任务派给编剧。
3. 编剧完成后调用 `org_submit_deliverable` 交付脚本。
4. 制片人验收脚本，随后派单给通义生图工作台。
5. 通义生图工作台调用 `tongyi_image_create`。
6. 通义生图工具返回 JSON，里面应包含 `image_urls`、`local_paths` 或 `asset_ids`。
7. 组织 runtime 自动把图片产物落到 `plugin_assets/` 目录并登记为附件。
8. 制片人把 `asset_ids` 或图片信息传给即梦视频工作台。
9. 即梦视频工作台调用 `seedance_create`，参数中应包含 `from_asset_ids`。
10. 即梦视频工具返回 JSON，里面应包含 `video_url`、`video_path`、`asset_ids` 或 `local_paths`。
11. 组织 runtime 自动把视频产物登记为附件。
12. 制片人最终向用户交付完整结果。

预期最终回复：

1. 有完整短片脚本。
2. 有 3 个镜头说明。
3. 能看到分镜图附件。
4. 能看到视频附件。
5. 回复中说明通义生图和即梦视频两个工作台都已参与。

验收重点：

1. 最终结果不能只有文字。
2. 附件区域应出现图片和视频。
3. 如果系统显示任务验收状态，不应是 `INCOMPLETE`。
4. 消息流中应出现工作台节点的任务交付记录。

### 5.2 只测通义生图工作台

用户输入：

```text
只测试通义生图工作台：请为「未来城市咖啡广告」生成 3 张关键帧分镜图。

每张图都要是 9:16 竖屏，赛博朋克雨夜风格：
1. 镜头一：雨夜街角，主角拿着发光咖啡杯，背景是霓虹招牌。
2. 镜头二：咖啡杯特写，杯身反射未来城市灯光。
3. 镜头三：主角站在高架天桥上，城市天际线和无人机灯光在背后。

请调用通义生图工作台完成，不需要生成视频。最终交付图片附件和每张图对应的 prompt 摘要。
```

预期结果：

1. 通义生图工作台被派单。
2. `tongyi_image_create` 被调用。
3. 最终交付中有图片附件。
4. 结果中包含 `asset_ids` 或至少说明图片已登记为附件。

### 5.3 只测即梦视频工作台（使用上一步 asset_ids）

前置：

从 5.2 的结果中复制通义生图返回的 `asset_ids`。如果页面没有展示 `asset_ids`，可以使用分镜图附件已登记的路径或让制片人从上一次交付中读取。

用户输入模板：

```text
只测试即梦视频工作台：请使用下面这些上游分镜图 asset_ids 生成一个 9:16 竖屏短视频。

asset_ids:
1. <把第一张图的 asset_id 粘贴在这里>
2. <把第二张图的 asset_id 粘贴在这里>
3. <把第三张图的 asset_id 粘贴在这里>

要求：
1. 必须调用 seedance_create。
2. 必须把这些 asset_ids 填入 from_asset_ids。
3. 视频风格保持赛博朋克雨夜、霓虹灯、电影感。
4. 最终交付视频附件，并说明是否生成 last_frame。
```

预期结果：

1. 即梦视频工作台被派单。
2. `seedance_create` 被调用。
3. 调用参数包含 `from_asset_ids`。
4. 最终交付中有视频附件。

### 5.4 负向测试：工作台缺失或未启用

操作：

1. 暂时禁用 `seedance-video` 插件。
2. 重启或刷新插件状态。
3. 再执行 5.1 的完整任务。

用户输入：

```text
请制作一个 15 秒竖屏短片，主题是「雨夜霓虹咖啡」。
请按脚本、分镜图、视频三个阶段完成，并调用通义生图工作台和即梦视频工作台。
```

预期结果：

1. 系统不应崩溃。
2. 通义生图阶段可以继续。
3. 视频阶段应给出明确错误或说明即梦视频工作台不可用。
4. 最终回复应提示需要安装或启用 `seedance-video` 工作台。

恢复：

1. 重新启用 `seedance-video` 插件。
2. 重新运行完整测试。

用户输入：


## 7. 路径三：IM 通道测试

IM 通道包括飞书、钉钉、Telegram、企业微信、QQ、OneBot 等。IM 是即时通讯入口，用户在外部聊天软件中向 OpenAkita 机器人发送消息，机器人再把消息转入 OpenAkita 后端。

### 7.1 IM 前置检查

1. 按 `docs/im-channel-setup-tutorial.md` 完成至少一个 IM 通道配置。
2. 在 OpenAkita 状态页确认该 IM 通道在线。
3. 在对应 IM 应用里找到机器人。
4. 如果是群聊，需要按平台规则 @ 机器人；如果是单聊，直接发送即可。

### 7.2 IM 基础连通性

用户在 IM 中发送：

```text
你好，确认你在线。请只回复「OpenAkita IM 在线」。
```

预期结果：

```text
OpenAkita IM 在线
```

### 7.3 IM 调用组织完整流程

用户在 IM 中发送：

```text
@org:AIGC 视频创作工作室测试 请制作一个 30 秒竖屏 AIGC 短片，主题是「未来城市咖啡广告」。

请按以下流程执行：
1. 编剧先写 3 个镜头脚本，每个镜头包含画面描述、字幕、镜头运动和时长。
2. 通义生图工作台根据每个镜头生成关键帧分镜图。
3. 把通义生图返回的 asset_ids 传给即梦视频工作台。
4. 即梦视频工作台必须调用 seedance_create，并使用 from_asset_ids。
5. 最终在 IM 里回复脚本摘要、图片附件信息、视频附件信息。

风格：赛博朋克、霓虹灯、雨夜、电影感，9:16 竖屏。
```

如果 IM 平台要求先 @ 机器人，则发送格式示例：

```text
@OpenAkita @org:AIGC 视频创作工作室测试 请制作一个 30 秒竖屏 AIGC 短片，主题是「未来城市咖啡广告」。
后续要求同上。
```

预期结果：

1. IM 中收到任务已开始或组织处理中的回复。
2. OpenAkita 组织页面能看到对应组织开始执行任务。
3. 最终 IM 中应收到脚本摘要和附件信息。
4. 如果平台支持文件/视频发送，应能直接收到图片或视频。
5. 如果平台不支持视频或本地文件发送，应收到文件名、路径或下载链接说明。

注意：

1. 企业微信智能机器人对视频/文件支持有限，可能只返回文本和图片。
2. QQ 官方机器人富媒体发送可能需要公网 URL；如果没有公网 URL，可能降级为文本链接或路径。
3. 飞书、钉钉、Telegram 通常更适合验收附件回传。

1. 制片人节点接收任务。
2. IM 收到阶段性或最终回复。
3. 组织页面同步出现任务流和附件登记。

### 7.5 IM 中断与追问测试

在完整任务执行中途，用户在 IM 中发送：

```text
请暂停一下，先告诉我目前完成到了哪一步：脚本、分镜图、视频分别是什么状态？
```

预期结果：

1. 系统能回复当前进度。
2. 已完成的脚本或分镜图不应丢失。
3. 如果任务仍在运行，应说明下一步会继续生成视频或等待工作台结果。

任务完成后，用户在 IM 中发送：

```text
请把最终结果重新整理成一个简短交付清单：脚本、每张分镜图、视频分别叫什么文件，分别在哪里。
```

预期结果：

1. 回复中列出脚本文件、图片文件、视频文件。
2. 如果 IM 平台不能直接发文件，也应给出文件名和可访问路径或下载提示。

## 9. 成功判定标准

### 9.1 完整流程成功

满足以下条件即可认为完整流程通过：

1. 用户只输入一次完整任务。
2. 制片人自动委派编剧、通义生图工作台、即梦视频工作台。
3. 编剧交付脚本。
4. 通义生图工作台交付图片。
5. 图片产物被自动登记为附件。
6. 制片人把 `asset_ids` 或等价资产引用传递给即梦视频工作台。
7. 即梦视频工作台使用 `from_asset_ids` 生成视频。
8. 视频产物被自动登记为附件。
9. 最终回复包含脚本摘要、图片附件、视频附件。
10. 没有出现 `INCOMPLETE` 或「缺少附件」类验收失败。

### 9.2 降级流程成功

如果某个 IM 平台不支持视频或文件直接发送，只要满足以下条件，也可认为 IM 降级路径通过：

1. 组织内实际已经生成并登记了视频附件。
2. IM 回复明确说明附件已生成。
3. IM 回复给出文件名、路径或可下载链接。
4. 没有因为平台限制导致组织任务失败。

## 10. 常见问题排查

### 10.1 模板里有工作台节点，但运行时提示工具不可用

可能原因：

1. `tongyi-image` 或 `seedance-video` 没安装。
2. 插件已安装但未启用。
3. 插件启用后没有重启后端，PluginManager 没加载到工具。
4. API Key 未配置，插件启动失败。

处理：

1. 打开「插件管理」确认插件状态。
2. 配置对应 API Key。
3. 重启 OpenAkita 后端。
4. 重新打开组织，确认工作台节点能力页中工具名仍然存在。

### 10.2 通义生图成功了，但最终没有图片附件

可能原因：

1. 插件返回的 JSON 中没有 `image_urls`、`local_paths` 或 `asset_ids`。
2. 远端图片下载失败。
3. 图片大小超过 runtime 限制。
4. 文件路径不在允许范围内，被安全校验拒绝。

检查：

1. 查看工具调用结果中是否有 `registered_attachments`。
2. 查看 org workspace 的 `plugin_assets/tongyi-image/` 目录是否有图片文件。
3. 查看日志中是否有下载失败或路径拒绝信息。

### 10.3 即梦视频没有使用上游图片

可能原因：

1. 制片人没有把 `asset_ids` 传给视频工作台。
2. 即梦视频工作台没有把 `asset_ids` 填入 `from_asset_ids`。
3. Asset Bus 中的资产过期或不可消费。
4. 插件缺少 `assets.consume` 权限。

检查：

1. 查看 `seedance_create` 的调用参数是否包含 `from_asset_ids`。
2. 查看 `seedance-video` 插件权限是否包含 `assets.consume`。
3. 查看工具返回 JSON 是否有 `video_url` 或 `video_path`。

### 10.4 IM 中收不到附件

可能原因：

1. IM 平台本身不支持该类型附件。
2. 文件需要公网 URL，但当前只生成了本地路径。
3. 机器人权限不足。

处理：

1. 先在 OpenAkita Web / Desktop 组织页面确认附件确实已生成。
2. 再判断是 IM 平台发送能力限制，还是组织内部没有生成附件。
3. 对不支持视频的平台，验收文本回传路径即可。
