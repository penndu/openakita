# G6 — API / channels swap (Phase 6)

**Status:** Signed off.

## Background

Phase 6 swaps the orchestration surface to the v2 stack, gated by
``settings.runtime_v2_enabled`` (still default ``False`` during the
canary window). Per ADR-0001 (fork-style rewrite) and ADR-0002, the
swap covers the backend HTTP facade, the channels gateway entry
point, and the frontend entry point.

## Residual risk

* The channels gateway hook is **observation-only**. Inbound IM
  messages still take the legacy path even when v2 is enabled. The
  Phase 7 cutover commit replaces the canary log with an actual
  ``runtime.supervisor.Supervisor.run`` call.
* The JSON-file backed org store is fine for a single-process FastAPI
  worker (canary mode) but does not survive multi-process deploys.
  Phase 7 swaps this for the SQLite-backed ``runtime.checkpoint``
  store; the HTTP contract on ``/api/v2/orgs`` stays the same.
* OrgChatPanel.tsx still only subscribes to the legacy event stream.
  Deferred to a post-Phase-7 frontend commit where the new event
  source (v2 supervisor) actually exists.
* ``TemplatePickerDrawer`` is not yet mounted into any view by
  default. Operators discover it via the v2-aware editor when Phase
  7 flips the flag.
