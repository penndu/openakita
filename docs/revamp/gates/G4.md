# G4 — Nodes (Phase 4)

**Status:** Signed off.

## Background

Phase 4 turns the runtime's abstract `Node` slot into a concrete
hierarchy of first-class node types, per ADR-0007 (node protocol)
and ADR-0009 (plugin workbench manifest). The state-graph router
(G3) was the previously-missing target for `ConditionNode`; with
that landed, every node type the templates use can be wired up.

## Delivered

| Module                               | ADR        | Notes                                                                                                       |
| ------------------------------------ | ---------- | ----------------------------------------------------------------------------------------------------------- |
| `runtime/nodes/base.py`              | ADR-0007   | `NodeProtocol` + `NodeContext` + `BaseNode` lifecycle state machine; defensive exception promotion; cooperative cancel routing. 13 tests. |
| `runtime/nodes/tool_node.py`         | ADR-0007   | Single-tool step with documented JSON / metadata argument parsing. 8 tests.                                  |
| `runtime/nodes/llm_node.py`          | ADR-0007   | Brain-driven node with bounded ReAct tool loop, allow-list rejection, budget guard, runner-absent recovery. 8 tests. |
| `runtime/nodes/condition_node.py`    | ADR-0007   | Deterministic sync/async predicate routing with construction-time validation; deterministic failure on unknown / non-string labels. 8 tests. |
| `runtime/nodes/human_review_node.py` | ADR-0007   | Three-verdict (APPROVE / REJECT / EDIT) review pause with cooperative cancel cleanup; `InMemoryReviewQueue` reference. 9 tests. |
| `runtime/nodes/manifest.py`          | ADR-0009   | `WorkbenchManifest` / `WorkbenchMode` / `WorkbenchUI` dataclasses + `parse(raw_dict)` validator. 14 tests.   |
| `runtime/nodes/workbench_node.py`    | ADR-0007 + ADR-0009 | Plugin-as-node with mode-scoped tool allow-list, explicit mode switching, lifecycle envelope (`workbench_ready` / `workbench_mode_switched` / `workbench_cancelled`). 9 tests. |
| `plugins/manager.py` (loader-side)   | ADR-0009   | Plugin loader now discovers the top-level `WORKBENCH` constant at load time and parses it through `manifest.parse`. 5 discovery tests.    |
| `plugins/happyhorse-video/plugin.py` | ADR-0009   | Four-role manifest (art_director / image_artist / video_animator / portrait_actor) + load-time validation in `tests/test_workbench_manifest.py`. |

## Residual risk

* `WorkbenchNode` cancellation depends on the plugin process
  cooperating; for non-cooperating plugins the supervisor still
  needs a hard kill path. Tracked but not blocking.
* `LLMNode` reads the Brain via the concrete legacy type; switching
  to `SupervisorBrain` Protocol (Phase 2 commit 16) is deferred to
  the Phase 6 channels-gateway swap.

## Sign-off

side manifest discovery in `plugins/manager.py`. G4 signed off.
