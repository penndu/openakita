# G5 — Templates: schema, registry, four builtins (Phase 5)

**Status:** Signed off.

## Background

Phase 5 ships a typed template registry plus four first-party
templates per ADR-0008. Templates are how v2 surfaces a usable
"clone an org from a recipe" experience; they replace the legacy
`orgs/templates.py` (1 234 LOC) and
`orgs/plugin_workbench_templates.py` (225 LOC). The Phase-5 audit
also includes the **parent_id-from-HIERARCHY fix** so
`OrgV2.root_nodes()` and `children_of()` return real trees.

## Verification evidence

* Phase 5 totals: **88 template tests** (incl. 5 new parent-id
  regression tests).
* `ruff check src/openakita/runtime/templates` — clean.
* The parent-id fix lands on `630035a1`.
* All four flagship template ids present in the registry after
  `discover_builtins() + GLOBAL_REGISTRY.bootstrap()`.

## Residual risk

* `GLOBAL_REGISTRY.bootstrap()` is not yet called by anything in
  production code — only by tests. The Phase-6 facade is responsible
  for wiring this at server startup (planned in the channels-gateway
  + `/api/v2/orgs/templates` integration).
* The Phase-5 templates assume the workbench plugin is present when
  the template is instantiated; if not, `WorkbenchNode` falls back to
  a degraded mode (logged) but no preflight check exists. Acceptable
  for now because templates are operator-curated.
