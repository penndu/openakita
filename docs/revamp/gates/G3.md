# G3 — Runtime engine: supervisor, messenger, guardrail, state graph (Phase 3)

**Status:** Signed off.

## Background

Phase 3 builds the orchestration engine of v2: a dual-ledger
supervisor that owns the outer loop, a messenger that resolves
addresses and cancels cooperatively, a guardrail runner that scores
deliverables, and the Pregel-style `StateGraph` that gives
`ConditionNode` somewhere to route to. Per ADR-0002 / ADR-0004 /
ADR-0007.

## Delivered

| Module                            | ADR       | Notes                                                                                                                |
| --------------------------------- | --------- | -------------------------------------------------------------------------------------------------------------------- |
| `runtime/ledger.py`               | ADR-0004  | `TaskLedger` + `ProgressLedger` + parser. Verdict taxonomy DONE / PROCEED / SUSPECT / REPLAN / OUT_OF_TURNS.        |
| `runtime/stall_detector.py`       | ADR-0004  | `n_stalls` regen logic; regression-guard tested with the long-progressing-storyboard fixture.                       |

| `runtime/guardrail/runner.py`     | ADR-0007  | `GuardrailRunner.evaluate` returning OK / RETRY / HARD_FAIL aggregated from validators.                              |
| `runtime/guardrail/builtin.py`    | ADR-0007  | Built-in validators: min/max length, required fields, regex pattern.                                                 |
| `runtime/state_graph.py`          | ADR-0002 + ADR-0007 | Pregel-style graph with `add_node` / `add_edge` / `add_conditional_edges` / `set_entry_point` / `validate`. `route()` resolves priority: conditional > static > delegation hint > defer-to-supervisor. `compile_from_org` projects an OrgV2 into the graph (HIERARCHY / COLLABORATE → static, CONSULT dropped). |

## Residual risk

* `StateGraph` is not yet consumed by the legacy ReasoningEngine —
  the v2 reasoning rewrite is a Phase-2 facade today and Phase 8
  picks up the StateGraph migration. Until then, the graph drives
  only the v2 supervisor's delegation routing.
* `Supervisor` depends on the concrete legacy Brain; the new
  `SupervisorBrain` Protocol (Phase 2 commit 16) is not yet wired
  into the supervisor's type seam. Low-risk technical debt; not
  blocking G3 because the integration tests still pass against the
  concrete type.
