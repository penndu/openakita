# G1 — Foundation runtime leaf modules (Phase 1)

**Status:** Signed off (paperwork).

## Background

ADR-0001 (fork-style rewrite) established that v2 lives in a new
`src/openakita/runtime/` package alongside the legacy `src/openakita/
core/` tree. G1 is the gate after Phase 1, in which the foundation
leaves of that runtime tree land: models, cancel tokens, retry policy,
stream bus, event store, checkpointers, ledgers, stall detector, etc.
None of these need to interact with the LLM yet, so G1 is the
"can we model the new world in pure Python" gate.

## Delivered

The following Phase-1 modules are shipped under `src/openakita/runtime/`:

* `models.py` — `OrgV2`, `NodeV2`, `EdgeV2`, `EdgeKind` (ADR-0002).
* `cancel_token.py` — `CancellationToken` + `CancelledByToken`.
* `retry_policy.py` — `RetryPolicy` + retriable taxonomy.
* `stream.py` — `StreamBus` + the 8 v2 channels (ADR-0006).
* `event_store.py` — hash-chained SQLite WAL log (ADR-0005).
* `checkpoint.py` — `BaseCheckpointer` + `MemoryCheckpointer`.
* `backends/sqlite.py`, `backends/json_file.py` — durable
  checkpointers.

## Residual risk

* `SqliteCheckpointer` has only an in-memory smoke test; real-FS
  durability ("crash-during-write") is not exercised. Risk is low
  because writes go through SQLite's WAL.
* `StreamBus` does not back-pressure slow subscribers; today every
  consumer is in-process. Documented; will be revisited when channels
  gateway lands in Phase 6.
