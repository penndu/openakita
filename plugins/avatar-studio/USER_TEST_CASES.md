# Avatar Studio 用户测试用例

> 版本：1.0.0 | 适用于 OpenAkita + DashScope API  
> 预估总耗费：¥2–5（全部用例跑完）  
> 预估总耗时：约 30–45 分钟（含 DashScope 排队等待）

---

## 〇 · 前置准备

### 0.1 DashScope API Key

1. 访问 [阿里云百炼控制台](https://bailian.console.aliyun.com/)，登录后进入「API-KEY 管理」
2. 创建或复制一个 API Key（`sk-` 开头），确保账户余额 ≥ ¥10
3. 在百炼控制台 → 「模型广场」中**逐一开通**以下模型（点"开通"即可，无需付费）：
   - `wan2.2-s2v`（图片说话）
   - `wan2.2-s2v-detect`（人脸预检）
   - `videoretalk`（视频换嘴）
   - `wan2.2-animate-mix`（视频换人）
   - `wan2.5-i2i-preview`（多图融合）
   - `cosyvoice-v2`（语音合成）
   - `qwen-vl-max`（多模态 LLM，可选）

### 0.2 测试素材（免费公开素材）

| 编号 | 素材 | 来源 / 获取方式 | 用途 |
|---|---|---|---|
| **IMG-01** | 女性正面半身照 | DashScope 官方示例：`https://help-static-aliyun-doc.aliyuncs.com/file-manage-files/zh-CN/20240829/lyumdf/female_2.png` — 右键另存为 `female_2.png` | 照片说话 / 形象库 / 数字人合成 |
| **IMG-02** | 男性正面半身照 | 在 [Unsplash](https://unsplash.com/s/photos/portrait-man-frontal) 搜索 "portrait man frontal"，选一张光线均匀、无墨镜、无遮挡的证件照风格图片，下载保存为 `male_portrait.jpg` | 视频换人 / 形象库 |
| **IMG-03** | 场景图（咖啡馆内景） | 在 [Unsplash](https://unsplash.com/s/photos/cafe-interior) 搜索 "cafe interior"，下载一张咖啡馆内景图保存为 `cafe_bg.jpg` | 数字人合成（场景参考图） |
| **VID-01** | 说话人视频（≤15 秒） | 在 [Pexels](https://www.pexels.com/search/videos/person-talking/) 搜索 "person talking"，下载一段 10-15 秒的正面讲话视频保存为 `talking.mp4`（建议选择人物正面、嘴部清晰的视频） | 视频换嘴 |
| **VID-02** | 动作人视频（≤15 秒） | 在 [Pexels](https://www.pexels.com/search/videos/person-walking/) 搜索 "person walking" 或 "person gesture"，下载一段 10-15 秒有身体动作的视频保存为 `action.mp4` | 视频换人 |
| **AUD-01** | 现成音频（可选） | 自己用手机录一段 5 秒的说话音频，保存为 `sample.mp3`；或从 [Freesound](https://freesound.org/) 下载一段人声 | 音色克隆 / 音频上传覆盖 TTS |

> **素材要求总结**：人像照必须是**真人、正面、单人、无遮挡**（无墨镜、口罩、厚刘海），否则 `wan2.2-s2v-detect` 预检会失败。视频建议 **≤ 30 秒、人物正脸占比 ≥ 60%**。

## 一 · Settings Tab（设置）— 最先测试

### TC-S01：首次配置 API Key

**前提**：插件刚加载，尚未配置 API Key

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 打开 Avatar Studio 插件 UI | 顶部 header 右侧出现**黄色** API Key 状态标签（显示"未配置 API Key"） |
| 2 | 点击顶部黄色标签 | 自动切换到 **Settings** Tab |
| 3 | 在「API Key」输入框粘贴你的 `sk-xxxx` | 输入框正常显示（密码类型可能隐藏） |
| 4 | 「服务地域」保持默认"北京" | 不需要改动 |
| 5 | 点击「测试连接」按钮 | 出现绿色提示"API Key 有效" |
| 6 | 点击页面底部「保存设置」按钮 | Toast 提示"设置已保存"；顶部标签变为**绿色**（显示"API Key"） |

### TC-S02：修改默认偏好

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | Settings → 「默认偏好」区域 | 看到分辨率下拉、默认音色下拉、成本阈值输入框 |
| 2 | 将「默认分辨率」改为 `720P` | 下拉切换成功 |
| 3 | 将「成本确认阈值」从 `5` 改为 `0.5` | 输入框内容变为 `0.5` |
| 4 | 保存设置 | Toast 提示保存成功 |
| 5 | 刷新页面 → 重新进入 Settings | 分辨率显示 `720P`，阈值显示 `0.5`（验证持久化） |
| 6 | **测试完后把阈值改回 `5`，分辨率改回 `480P`** | 还原默认，避免影响后续用例 |

### TC-S03：填写错误的 API Key

| 步骤 | 操作 | 预期结果 |
|---|---|---|

## 五 · Tasks Tab（任务列表）

### TC-T01：查看任务列表

**前提**：已在 TC-C01 等用例中成功提交过任务

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 切换到 **Tasks** Tab | 看到任务列表，每行包含：模式图标、任务 ID（短码）、状态徽章、创建时间、费用、操作按钮 |
